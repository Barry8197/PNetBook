Reactome Pathway Network Module

reactome.add_edges(G, node, n_levels)[source]

Adds sequential edges to a graph, creating a chain of nodes extending from an initial node.

The function constructs a series of edges starting from a given node, appending increasing levels to the node name to create subsequent nodes in the sequence. Each node is connected sequentially to the next, forming a linear progression of edges. These edges are then added to the graph.

Parameters:

  • G (networkx.Graph) – The graph to which the edges will be added. This graph must support the add_edges_from method as found in NetworkX graphs.

  • node (str) – The base name of the initial node from which to extend the edges.

  • n_levels (int) – The number of additional levels (i.e., nodes) to create and connect starting from the initial node. n_levels of 1 would mean one additional node connected to the original.

Returns:

The graph with the new edges added, linking the series of nodes created from the base node as described.

Return type:

networkx.Graph

Example

>>> import networkx as nx
>>> G = nx.Graph()
>>> G = add_edges(G, 'base_node', 3)
In this example, edges are added to connect 'base_node' to 'base_node_copy1',
'base_node_copy1' to 'base_node_copy2', and 'base_node_copy2' to 'base_node_copy3'.
reactome.complete_network(G, n_leveles=4)[source]

Completes a network from a specified root node to ensure all paths extend to a specified depth.

This function extends the paths in a graph starting from a ‘root’ node to make each terminal path reach a specific depth, measured in number of levels. Terminal paths shorter than the requested depth are extended accordingly.

Parameters:

  • G (networkx.DiGraph or networkx.Graph) – The graph on which operations will be performed. This graph should be directed to correctly calculate out-degrees and path lengths.

  • n_leveles (int, optional) – The desired minimum depth of all paths starting from the ‘root’ node. Default value is 4.

Returns:

A subgraph of G containing nodes within the specified radius from ‘root’ and extended paths to match the desired levels if they are originally shorter.

Return type:

networkx.Graph

Notes

  • The paths are not extended if they already meet or exceed the number of ‘n_leveles’.

  • It is assumed there are no cycles from ‘root’ to any of its descendants since this would imply infinite paths upon attempting to reach the depth specified by ‘n_leveles’.

Example

>>> import networkx as nx
>>> G = nx.DiGraph()
>>> G.add_edge('root', 'level_1')
>>> G.add_edge('level_1', 'level_2')
>>> completed_G = complete_network(G, n_leveles=5)
This modifies the graph by potentially adding additional nodes and edges to ensure that
each path from 'root' reaches a total depth of 5.
reactome.get_layers_from_net(net, n_levels)[source]

Extracts node information layer by layer up to a specified depth from a given network.

This function traverses a network structure at given depths (levels), extracting the nodes at each level and their immediate successors. The function focuses on the original name of nodes by removing any suffixes that indicate copies or duplications, grouping the successors similarly.

Parameters:

  • net (networkx.DiGraph) – The directed graph from which nodes and their relationships are extracted. The graph is expected to have nodes potentially labeled with suffixes indicating layers or copies.

  • n_levels (int) – The number of levels to traverse in the net structure, starting from level 0, which is often considered the root or initial level.

Returns:

A list of dictionaries, where each dictionary represents a layer in the network.

The keys of the dictionary are the node names (with copy suffixes removed), and the values are lists of immediate successor node names (also with copy suffixes removed).

Return type:

List[dict]

Example

>>> import networkx as nx
>>> net = nx.DiGraph()
>>> net.add_edge('root', 'node1_copy1')
>>> net.add_edge('root', 'node2_copy1')
>>> net.add_edge('node1_copy1', 'node3_copy2')
>>> layers = get_layers_from_net(net, 2)
>>> print(layers)
[{'root': ['node1', 'node2']}, {'node1': ['node3']}]

Note

This function handles networks where nodes may have copies or are duplicated at various levels, and names contain suffixes indicating such duplication. This results from the fact that pathways can be included in more than 1 level if they are involved in different levels of different pathways
